## Java / Backend Rules

- Java 21, no Lombok. Use records for DTOs and repository rows.
- jOOQ **plain DSL only** ‚Äî `table("name")`, `field("name")`. No code generation.
- No JPA / Hibernate. All persistence via `DSLContext`.
- Flyway migrations: sequential V1, V2, ‚Ä¶ naming. Never edit existing migrations.
- Spring Security: stateless JWT. All endpoints require auth except `/api/auth/**`.
- Custom exceptions extend `DomainException`. Map to HTTP status in `GlobalExceptionHandler`.
- Repository pattern: interface defines `Row` record + method signatures; jOOQ impl in `repository/jooq/`.
- Services inject repositories via constructor. No `@Autowired` field injection.
- OpenAI calls through `OpenAiClient` only. Never expose API key to frontend.
- CV files stored via `FileStorageService` interface (local implementation). Abstraction allows future S3 swap.
- Scheduled jobs annotated with `@Scheduled`. `@EnableScheduling` on main class.

## TypeScript / Frontend Rules

- React 18 functional components. No class components.
- All server state via TanStack Query (`useQuery`, `useMutation`). No `useState` for async data.
- Client-only state (auth tokens) via Zustand persisted to localStorage.
- Tailwind CSS only ‚Äî no CSS modules, no inline styles.
- All API calls through `apiClient` (Axios instance in `app/api.ts`).
- TypeScript strict mode. All interfaces in `src/types/index.ts`.
- Toast notifications via `sonner`. 
- Forms with `react-hook-form` + `zod` validation.
- Use `cn()` from `lib/utils` for conditional class names.
- Error messages via `getErrorMessage(err)` from `app/api.ts`.

## Product Rules

- **No scraping.** All job intake is manual ‚Äî users paste JDs and source details.
- JD analysis runs before adding to pipeline ‚Äî returns APPLY/MAYBE/SKIP recommendation.
- Never hallucinate about companies (visa, relocation, salary) unless source data confirms it.
- All messages generated by AI must pass guardrails: no desperation, no overexplaining visa, no weak salary language.
- Positioning always: Senior Backend Engineer ‚Äî payments, fintech, distributed systems, reliability, production ownership.
- Salary positioning: ‚Ç¨80k‚Äì95k practical, ‚Ç¨95k‚Äì110k strong fit. Never below ‚Ç¨75k Munich.

## Coding Style

- Record constructors for DTOs: `record FooResponse(String id, String name) {}`
- `Optional<T>` for nullable returns from repositories.
- `UUID` for all entity IDs.
- Log at `INFO` for business events, `DEBUG` for verbose tracing, `WARN` for degraded paths.
- No System.out. Use `Logger log = LoggerFactory.getLogger(...)`.
- `@Transactional` at service method level when multiple writes must be atomic.
